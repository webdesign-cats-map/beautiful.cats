# 校园流浪猫 WebGIS 系统 - 需求分析文档

**版本**：v1.0  
**编写日期**：2024年

---

## 一、项目背景

校园内存在大量流浪猫，师生对其关注度高，但缺乏统一的管理和信息共享平台。本系统通过 WebGIS 技术，将流浪猫的地理位置、档案信息、打卡记录等进行可视化管理，实现师生共同参与的校园流浪猫关爱生态。

---

## 二、用户角色

| 角色 | 权限说明 |
|------|----------|
| 普通用户 | 查看地图、档案、打卡记录；上报位置与猫咪信息；偶遇/投喂打卡；发表评论 |
| 管理员 | 拥有普通用户全部权限；审核上报内容；维护猫咪档案；管理用户权限 |

---

## 三、功能需求

### 3.1 基础功能

#### 3.1.1 WebGIS 底图集成

- 接入高德地图 JS API 2.0，展示校园 2D 底图
- 支持缩放（鼠标滚轮/手势）与平移（拖拽）
- 支持标准/卫星图层切换
- 支持 GPS 定位当前位置
- 地图选点功能（用于打卡和上报）

#### 3.1.2 用户与权限管理

- 用户注册、登录、登出
- 用户信息查看与修改
- 角色区分（普通用户/管理员）
- 管理员可升级普通用户为管理员

#### 3.1.3 校园猫咪档案库

猫咪档案包含：
- 基本信息：ID、昵称、性别、毛色
- 状态信息：健康状态（健康/受伤）、绝育状态
- 标签信息：个性标签（亲人/社恐/活泼等）
- 位置信息：出没坐标 + 地址描述
- 统计信息：打卡次数、最近出没时间
- 描述信息：猫咪介绍

#### 3.1.4 地图点位查看

- 以校园地图为底图，标注流浪猫常出没点位
- 自定义 Marker 样式（猫咪图标）
- 点击点位弹出猫咪信息卡片
- 信息卡展示基本信息、标签、位置
- 支持按性别、健康状态筛选点位
- 支持名称关键词搜索 + 地图飞行定位

#### 3.1.5 偶遇/投喂打卡

- **偶遇打卡**：记录发现猫咪的位置和时间
- **投喂打卡**：记录投喂信息，可关联到具体猫咪
- 支持 GPS 自动定位或手动输入坐标
- 支持上传照片
- 支持填写备注信息
- 打卡内容需经管理员审核后生效
- 审核通过后自动更新猫咪出没位置和打卡计数
- 打卡界面内嵌科学养猫科普提示

#### 3.1.6 故事墙与评论区

- 每只猫的详情页下方设有留言板
- 师生可发布与该猫的互动故事
- 显示发布者昵称和发布时间
- 评论按时间倒序展示

#### 3.1.7 个人中心

- 查看个人信息（用户名、昵称、角色）
- 我的收藏（已收藏的猫咪）
- 我的打卡记录（含审核状态）
- 猫咪图鉴（已发现/未发现的猫咪徽章）

### 3.2 进阶功能

#### 3.2.1 行踪可视化

- **热力图**：整合打卡数据，生成全局出没热力图，标注高频区域
- 热力图颜色渐变（蓝→绿→黄→红，对应低→高频次）

### 3.3 特色互动功能

#### 3.3.1 猫咪图鉴

- 用户每次审核通过的打卡对应一只猫咪的"发现"记录
- 个人中心展示猫咪图鉴，已发现的猫咪徽章点亮，未发现的置灰

#### 3.3.2 排行榜

- **明星猫咪榜**：按打卡次数排序，展示 Top3
- **活跃用户榜**：按打卡次数排序，展示 Top5

#### 3.3.3 数据看板

- 猫咪总数、绝育数量、需关注数量
- 打卡总数、待审核数、投喂次数
- 近7天打卡趋势折线图
- 猫咪性别分布饼图
- 猫咪健康状态饼图

### 3.4 管理后台

- **猫咪管理**：增删改查猫咪档案，设置位置坐标
- **打卡审核**：逐条审核用户打卡，通过/拒绝
- **上报审核**：审核用户上报的新猫咪信息
- **用户管理**：查看用户列表，调整用户角色

## 四、非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 地图加载时间 < 3秒；页面交互响应 < 500ms |
| 兼容性 | 支持 Chrome、Firefox、Edge 现代浏览器 |
| 安全性 | 密码哈希存储；接口权限校验 |
| 可维护性 | 前后端分离架构；代码注释完整 |
| 可扩展性 | 数据层可从localStorage无缝迁移到PostgreSQL |

---

## 五、技术选型

| 层次 | 技术 | 理由 |
|------|------|------|
| 前端框架 | 原生 HTML/CSS/JS | 轻量，无需构建工具，易于部署 |
| 地图服务 | 高德地图 JS API 2.0 | 国内覆盖好，API完善，校园地图数据准确 |
| 数据可视化 | Chart.js 4.x | 轻量，API简单，图表类型丰富 |
| 后端框架 | Python Flask 3.0 | 轻量级，适合中小型项目，开发效率高 |
| 数据存储（演示）| localStorage | 无需安装数据库，开箱即用 |
| 数据库（生产）| PostgreSQL + PostGIS | 支持空间数据，GIS行业标准 |

## 六、数据流程

```
用户打卡上报
    ↓
前端收集：位置坐标 + 照片 + 备注
    ↓
存储到 localStorage（待审核状态）
    ↓
管理员后台审核
    ↓
审核通过 → 更新猫咪位置和打卡计数 → 热力图数据更新
审核拒绝 → 标记为已拒绝
```
